## Task 1: Ch2 知识表示 — 实体发现方法论演进

### 版本 1：手工扩充（废弃）
直接将 schema 从 21 实体手工扩充到 44 实体。问题：缺乏方法论支撑，无法体现课程中的知识发现方法。

### 版本 2：词频 + jieba TF-IDF（废弃）
用 `jieba.analyse.extract_tags()` 做 TF-IDF 关键词提取。问题：只有 2 个全文本文档，TF-IDF 的 IDF 分量几乎不工作——因为几乎所有词都出现在两个文档中，IDF 值趋于零。结果中大量通用词（"机器"、"问题"、"工作"）排在专名前面。

### 版本 3：段落级 TF-IDF + POS + 共现增强（当前）
**核心洞察**：虽然语料只有 2 本书，但可以将每本书按段落切分（`\n\n` 分隔），得到 366 个段落。每个段落作为 TF-IDF 的一个"文档"，IDF 就能有效工作了——出现在大多数段落的词（如图灵、密码）IDF 低，只出现在特定段落的词（如哥德尔、贝尔实验室）IDF 高。

**方法组合**：
1. **段落级 TF-IDF**（sklearn TfidfVectorizer, max_df=0.6, min_df=2）：权重 0.3
2. **jieba 词性标注**（nr=人名, ns=地名, nt=机构名, nz=专名）：权重 0.4
3. **实体共现增强**（在已知实体 ±8 词窗口内出现的未识别词获得 boost）：权重 0.2
4. **词频归一化**：权重 0.1

**停用词**：采用 [goto456/stopwords](https://github.com/goto456/stopwords) 的 cn_stopwords + hit_stopwords 合并（993 词），替代手写列表。下载到 `data/` 目录，`entity_discovery.py` 启动时自动加载。

**效果**：TF-IDF 从近乎随机变为有意义的排序。英国（0.035）、美国（0.025）、德国（0.025）合理排在前面。"弗劳"（Flowers）、"米奇"（Michie）、"哥德尔"等真实专名被成功识别。

### 为什么不用纯聚类的方案？
- 中文文本需要先分词，分词结果中的人名片段（如"弗劳"、"米奇"）长度短、词频低，直接聚类容易和噪声混在一起
- POS 词性标注是 jieba 预训练的能力，比无监督聚类更可靠
- 共现增强利用已知实体做远程监督，相当于弱监督学习而非完全无监督

### 为什么混合人工审核？
- 知识工程的核心原则：机器辅助发现 + 人类专家验证
- 纯自动方法会产生大量假阳性（"东西"、"论文"被 POS 误标为专名）
- 人工审核体现了课程中"知识体系构建"的方法论

## Task 2: Ch4 实体识别 — 四方法对比

### 四种方法统一接口

每种 NER 方法均实现 `extract(sentences, gazetteer_mentions) -> CrfResult`：

| 方法 | 类型 | 实现 | 提及数 |
|------|------|------|--------|
| Gazetteer | 词典匹配 | 最长匹配 + 非重叠选择 | 8,861 |
| CRF | 判别式 | sklearn-crfsuite 线性链 CRF | 7,751 |
| HMM(手写) | 生成式 | 纯 Python π/A/B + 维特比 + 平滑 | 9,893 |
| HMM(hmmlearn) | 生成式 | CategoricalHMM + Baum-Welch EM | 436 |

### 手写 HMM 实现要点
- 三要素：初始概率 π（句子首个标签分布）、转移矩阵 A（BIO 标签转移）、发射矩阵 B（P(word|tag)）
- 维特比算法解码最优标签序列
- 拉普拉斯平滑（α=0.01）处理未登录词
- 训练数据：词典弱监督标签（与 CRF 一致）

### hmmlearn 探索教训
- v1：用原始词 ID 做观测→观测空间 ~10k，模型过拟合（31万提及）
- v2：改为特征桶化（字类型+长度桶，~45 观测类型）→信息损失严重（436 提及）
- 结论：hmmlearn CategoricalHMM 设计观测空间需精确平衡——太大过拟合，太小欠拟合

### 方法对比意义
HMM（生成式）与 CRF（判别式）对比直接体现了课程 Ch4 的核心理论：
HMM 建模 P(X,Y) 联合分布，可以生成样本；CRF 建模 P(Y|X) 条件分布，专注分类边界。

## Task 3: Ch4 开放域实体发现 — 打破封闭世界

### 问题诊断
前两个 task 之后，图谱节点数停滞在 28 个（关系中实体）。根因：整个流水线是封闭世界假设——NER 只能识别 schema 预定义的 46 个实体，Linking 只能链接到这 46 个。做再多 NER 方法优化也只是反复确认已知实体。

### 实现方案
在 `run_ner_stage` 内嵌开放域发现，不改 pipeline 阶段数：
1. 用 `entity_discovery.py`（段落 TF-IDF + POS + 共现）从文本发现候选
2. 高置信度候选（≥0.60）动态加入临时扩展 schema
3. 对新实体做第二轮 gazetteer 识别
4. 合并所有 mention

### 效果
- 发现 2,926 个新实体提及（22 独特实体）
- 9 个新实体进入三元组：哥德尔、达尔文、牛津、米奇、英国等
- 关系中实体：28 → 38（+36%）
- 三元组：107 → 119（+11%）

### 局限
- 仍有噪声（"东西"、"论文"被误标为专名）
- 多数发现实体无法形成关系（关系抽取仍然太弱）
- 需要更好的 POS 标注或弱监督过滤
